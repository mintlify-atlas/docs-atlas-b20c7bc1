Unikraft's core libraries provide the fundamental building blocks for unikernel construction. These internal libraries implement essential system services, abstractions, and utilities that higher-level applications and external libraries depend on.

## Common Patterns

### Registration and Initialization

Most Unikraft libraries follow a consistent initialization pattern:

1. **Early initialization** - Platform-specific setup via `earlytab`
2. **Library constructors** - C++ style constructors for library init
3. **Boot sequence** - Coordinated initialization through `ukboot`

### Abstraction Layers

Unikraft uses abstraction layers to separate interface from implementation:

- **Allocation**: `ukalloc` provides the interface, `ukallocbbuddy` implements it
- **Scheduling**: `uksched` defines the API, `ukschedcoop` provides an implementation
- **File Systems**: `vfscore` and `ukfs` provide abstractions for multiple FS types

### Configuration

All libraries are configured through KConfig options in their respective `Config.uk` files. Configuration options typically include:

- Feature enables/disables
- Buffer sizes and limits
- Debug and statistics options
- Implementation choices

## Using Core Libraries

Core libraries are automatically included based on dependencies specified in your application's `Config.uk`. For example:

```kconfig
config MYAPP
    bool "My Application"
    select LIBUKBOOT
    select LIBUKALLOC
    select LIBUKSCHED
```

The build system resolves dependencies and ensures libraries are initialized in the correct order.
